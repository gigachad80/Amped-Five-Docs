### 2.6 EDIT MODULE (Geometric & Distortion Correction)
**Purpose**: Geometric transformations, optical corrections, image preparation

#### **Filters & Sub-Filters**:

**Resize**
- **What it does**: Changes image dimensions with standard interpolation
- **When to use**: General resizing, but NOT for fingerprints
- **Key parameters**: Target width/height, interpolation method (bilinear/bicubic)
- **Situation**: Standard image resizing where precision isn't critical
- **Example**: Reduce file size - "Resize → 1920×1080 to 1280×720 → Reduces file size for faster processing"

**Resize 1:1 (Fingerprint-Specific)**
- **What it does**: Preserves original pixel count and DPI metadata without interpolation
- **When to use**: ALWAYS for fingerprints - AFIS compatibility requirement
- **Key parameters**: None - preserves exact original dimensions and DPI
- **Situation**: Fingerprint evidence preparation for AFIS comparison
- **Example**: Fingerprint from crime scene - "Resize 1:1 → Preserves 1000 DPI requirement → No interpolation artifacts → Ready for AFIS submission"
- **Critical requirement**: AFIS systems require minimum 1000 DPI and exact pixel preservation
- **NEVER use Smart Resize for fingerprints**: Will create interpolation that invalidates AFIS comparison

**Smart Resize**
- **What it does**: Intelligent upscaling using advanced interpolation
- **When to use**: License plates, text, non-fingerprint images requiring resolution increase
- **Key parameters**: Target size, interpolation algorithm (Lanczos/Cubic)
- **Situation**: Small license plate image requiring magnification before character recognition
- **Example**: Distant license plate - "Smart Resize → Upscale from 80×20px to 320×80px (4×) → Apply before sharpening for better character legibility"
- **Workflow position**: Apply BEFORE sharpening for best results
- **Advantage over standard Resize**: Superior edge preservation and detail reconstruction

**Crop**
- **What it does**: Removes unwanted areas, isolates region of interest (ROI)
- **When to use**: Focus analysis on specific region, reduce file size, remove irrelevant areas
- **Key parameters**: ROI coordinates, aspect ratio preservation
- **Situation**: Face in corner of wide shot, isolate footwear from scene, extract license plate region
- **Example**: Facial identification - "Crop → Select facial region 400×500px → Isolates face from busy background → Reduces processing time 90%"
- **Benefits**: Reduces file size, focuses enhancement, speeds processing

**Rotate**
- **What it does**: Rotates image by specified angle
- **When to use**: Correct skew, align images, standardize orientation
- **Key parameters**: 
  - **Fixed**: 90°, 180°, 270° (lossless)
  - **Free-form**: Custom angle with interpolation
- **Situation**: Tilted document photo, sideways video, alignment correction
- **Example**: Document analysis - "Rotate 3.2° clockwise → Corrects scanning skew → Text lines now horizontal"

**Undistort (Barrel/Pincushion)**
- **What it does**: Removes lens distortion from standard lenses
- **When to use**: Wide-angle lenses, measurement preparation, straight line correction
- **Key parameters**: 
  - **Distortion type**: Barrel (lines bow outward) or Pincushion (lines pinch inward)
  - **Calibration**: Camera focal length or manual line selection
- **Situation**: Wide-angle security camera with curved straight lines
- **Example**: Measurement prep - "Undistort → Register 3 straight reference lines (door frame, floor, ceiling) → Corrects barrel distortion → Ready for accurate measurement"
- **Measurement impact**: 5-15% dimensional error if uncorrected
- **Critical for**: Any measurement workflow (MUST correct before Measure 3D)
- **Workflow position**: After deblur, BEFORE measurement

**Correct Fisheye**
- **What it does**: Unwarps fisheye/dome camera extreme distortion
- **When to use**: Circular fisheye lenses, dome cameras, 180°+ field of view
- **Key parameters**: 
  - Reference line registration (minimum 3 curved lines)
  - Post-projection X/Y/Z shifts for recentering
- **Situation**: Dome camera ceiling view, fisheye security camera, ultra-wide lens
- **Example**: Dome camera height measurement - "Correct Fisheye → Click 3 curved lines (door frame edges, wall corners) → Unwarp from 180° circular to 120° linear → Post Projection shifts to recenter → Measurement accuracy <5% error"
- **Result**: ~180° circular view becomes 120-140° linear perspective
- **Accuracy**: <5% distortion achievable with proper calibration
- **Critical before**: ANY measurement workflow
- **Measurement impact**: ±30cm error without correction vs. ±1.3cm with correction

**Perspective Correction**
- **What it does**: Removes perspective distortion (keystone, trapezoidal skew)
- **When to use**: Camera tilted up/down (keystoning), off-angle documents, vanishing point distortion
- **Key parameters**: 
  - 4-point correspondence (corners of rectangle)
  - Reference plane selection
- **Situation**: Document photo at angle, license plate viewed from side, building facade from ground level
- **Example**: Off-angle document - "Perspective Correction → Click 4 document corners → Software calculates geometric transform → Rectangular document now frontal view"
- **Measurement impact**: 20-40% dimensional error if uncorrected
- **Must use before**: Measurement workflows, document analysis
