### 2.3 ANALYZE MODULE
**Purpose**: Verify integrity, understand source properties, diagnose quality

#### **Filters & Sub-Filters**:

**Hash Code**
- **What it does**: Computes cryptographic hash (MD5, SHA-1, SHA-256) of file
- **When to use**: Beginning of analysis AND after export (documentation)
- **Key parameters**: Hash algorithm selection (SHA-256 recommended)
- **Situation**: Chain of custody documentation for court admissibility
- **Example**: Evidence documentation - "Load original.mp4 → Hash Code → SHA-256: a3f5b2c... → Document in case file → After enhancement → Export enhanced.mp4 → New hash b7e4d9f... → Proves two different files, original preserved"
- **Legal importance**: Court can independently verify file hasn't been tampered with
- **Best practice**: Always generate hash BEFORE processing begins

**Metadata Analysis**
- **What it does**: Extracts and displays embedded file metadata
- **When to use**: Source verification, timestamp validation, camera identification
- **Key parameters**: Metadata type (EXIF/GPS/Codec/Technical)
- **Situation**: Verify recording device, capture timestamp, GPS location
- **Example**: Phone video evidence - "Metadata Analysis → EXIF shows: iPhone 13 Pro, captured 2024-01-15 14:32:17, GPS: 40.7128°N 74.0060°W (NYC) → Verifies device, time, location"
- **EXIF data (photos)**: Camera model, lens, ISO, aperture, shutter speed, GPS coordinates, timestamp
- **Video metadata**: Creation date, duration, codec (H.264/HEVC), bitrate, frame rate, resolution
- **Forensic value**: Proves authenticity, establishes timeline, identifies source device

**Histogram Tool**
- **What it does**: Displays tonal distribution graph (0-255 brightness levels)
- **When to use**: Before enhancement to assess exposure, guide adjustment decisions
- **Key parameters**: Channel selection (Luminance/RGB), region of interest
- **Situation**: Determine if image is underexposed, overexposed, or well-balanced
- **Example**: Dark facial image - "Histogram Tool → Shows pixel cluster at 20-40 range (left side) → Indicates severe underexposure → Decision: Apply Exposure adjustment first"
- **Interpretation guide**:
  - **Clustered left**: Dark image → use Exposure increase
  - **Clustered right**: Bright image → reduce Exposure
  - **Gaps/steps**: Posterization or limited color depth
  - **Even distribution**: Good dynamic range
  - **Clipping left/right**: Pure black/white loss of detail
- **Workflow use**: Check histogram BEFORE and AFTER each major adjustment

**Macroblocks Visualization**
- **What it does**: Displays H.264/MPEG compression block structure
- **When to use**: Assess video compression severity, plan deblocking strategy
- **Key parameters**: Visualization overlay intensity, block size display
- **Situation**: Compressed DVR footage with visible blocking artifacts
- **Example**: Compressed CCTV - "Macroblocks → Shows 16×16 block grid with high color variation between blocks → Decision: Apply Deblocking filter strength 0.7"
- **Technical detail**: H.264 compresses video in 8×8 or 16×16 pixel blocks; heavy compression creates visible block boundaries
- **Decision support**: Color variation magnitude indicates deblocking strength needed
- **Quality assessment**: Dense, colorful blocks = severe compression; sparse blocks = lighter compression
