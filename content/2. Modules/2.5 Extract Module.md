### 2.5 EXTRACT MODULE
**Purpose**: Reveal hidden detail, separate components, edge detection

#### **Filters & Sub-Filters**:

**Threshold**
- **What it does**: Converts to binary (black/white) using single threshold value
- **When to use**: High-contrast images, uniform backgrounds, document text
- **Key parameters**: Threshold value (0-255)
- **Situation**: Black text on white paper, silhouettes, stark contrast scenes
- **Example**: Document enhancement - "Threshold value 128 → All pixels below 128 become black, above 128 become white → Clear binary text extraction"
- **Limitation**: Requires uniform lighting; fails on varied backgrounds
- **When NOT to use**: Non-uniform backgrounds (use Adaptive Threshold instead)

**Adaptive Threshold**
- **What it does**: Local threshold calculation - adjusts threshold per image region
- **When to use**: Non-uniform lighting, varied backgrounds, complex substrates
- **Key parameters**: Block size (30×30 to 100×100 pixels typical)
- **Situation**: Fingerprint on textured wall, text on uneven paper, impression on varied surface
- **Example**: Fingerprint on painted wall - "Adaptive Threshold → Block size 75×75 → Each region gets custom threshold → Ridge detail emerges despite wall texture"
- **Advantage over Threshold**: Handles local lighting variations automatically
- **Block size guidance**: Larger blocks (100×100) for gradual lighting changes; smaller blocks (30×30) for abrupt changes

**Sobel Edge Detection**
- **What it does**: Computes gradient magnitude to highlight edges
- **When to use**: General edge finding, fast processing needed
- **Key parameters**: Kernel size (3×3 standard), direction (horizontal/vertical/both)
- **Situation**: Quick edge map generation for geometric analysis
- **Example**: Tool mark analysis - "Sobel → Highlights edges of impression → Creates clean edge map for measurement"
- **Characteristics**: Simpler and faster than Canny
- **Trade-off**: Speed vs. precision - use when speed is more important than perfect edge detection

**Canny Edge Detection**
- **What it does**: Multi-stage edge detection producing thin, well-connected edge lines
- **When to use**: Precision edge extraction for measurement, high-quality edge maps
- **Key parameters**: 
  - **Lower threshold** (50-100): Weak edge sensitivity
  - **Upper threshold** (150-200): Strong edge requirement
  - **Sigma** (1.0-2.0): Gaussian blur pre-processing
- **Situation**: Precise dimensional measurement of objects, geometric analysis
- **Example**: Weapon dimension measurement - "Canny → Lower 70, Upper 180, Sigma 1.5 → Clean edge outline → Use for Measure 2D perimeter calculation"
- **Advantage**: Removes false edges through multi-stage processing
- **When to use**: Edge quality is critical for subsequent measurement or analysis
- **Computational cost**: Higher than Sobel, but produces superior results

**Channel Mixer**
- **What it does**: Selects single RGB channel with best detail for analysis
- **When to use**: Different channels contain different information quality
- **Key parameters**: Channel selection (Red/Green/Blue), optional weights
- **Situation**: Forensic evidence where one color channel contains superior detail
- **Example - Fingerprints/Faces/Documents**: "Channel Mixer → Select GREEN channel → Most detail in faces, documents, and prints (human eye most sensitive to green wavelengths)"
- **Example - Bloodstains/Bruises**: "Channel Mixer → Select RED channel → Blood and bruises show maximum contrast"
- **Example - Shadows/Specific dyes**: "Channel Mixer → Select BLUE channel → Sometimes useful for shadows and specific dyes (rarely primary choice)"
- **Best practice**: Preview all three channels, select one with maximum contrast and detail
- **Speed**: Fastest extraction method - simple channel selection

**Color Deconvolution**
- **What it does**: Separates overlapping colors by creating custom color space
- **When to use**: Inks, stamps, graffiti, colored text on colored backgrounds
- **Key parameters**: 
  - Sample size (5-10 pixels)
  - Color space (Lab often superior to RGB)
  - Desired color selection
  - 2 undesired color selections
- **Situation**: Signature under red stamp, yellow text on blue wall, black text on yellow license plate
- **Example - Yellow text on blue wall**:
  - Step 1: Click yellow text pixel (target color)
  - Step 2: Click blue wall pixel (undesired color #1)
  - Step 3: Click background/substrate (if different)
  - Step 4: Optional - click second interfering color
  - Result: Yellow text isolated, blue wall removed
- **Example - Signature under stamp**: "Click signature ink (black) → Click stamp ink (red) → Stamp removed, signature visible"
- **Capability**: Can remove up to 2 undesired colors simultaneously
- **Color space selection**: Lab color space often provides better separation than RGB
- **Workflow position**: After restoration, before enhancement

**Fourier Filter (Frequency Domain)**
- **What it does**: Removes periodic/repeating patterns by frequency suppression
- **When to use**: ONLY for periodic patterns - chain-link fence, grid, repeating texture
- **Key parameters**: Frequency component selection, suppression strength
- **Situation**: Face behind chain-link fence, window grid obscuring evidence, wallpaper pattern interference
- **Example - Chain-link fence**: 
  - View frequency spectrum
  - Identify peak corresponding to fence pattern
  - Suppress that frequency component
  - Inverse transform back to spatial domain
  - Result: Fence pattern removed, face visible
- **Critical limitation**: ONLY works for periodic patterns, NOT random noise
- **When NOT to use**: Random noise (use Bilateral/Median filters instead)
- **Technical process**: Transforms image from spatial domain (pixels) to frequency domain (periodic components), suppresses unwanted frequencies, transforms back

**Component Separation**
- **What it does**: Advanced multi-component color separation
- **When to use**: Multiple overlapping colored elements requiring separation
- **Key parameters**: Number of components, separation method
- **Situation**: Complex color separation beyond 2-color Color Deconvolution
- **Example**: "Three overlapping ink colors on document → Component Separation → Extracts each ink color to separate layer"
