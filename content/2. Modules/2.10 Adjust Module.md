### 2.10 ADJUST MODULE (Tone & Color Correction)
**Purpose**: Brightness, contrast, exposure, and color enhancement

#### **Filters & Sub-Filters**:

**Brightness**
- **What it does**: Uniform increase/decrease of all pixel values
- **When to use**: Quick preliminary adjustment only
- **Key parameters**: Brightness offset (-100 to +100)
- **Situation**: Fast brightness change for preview
- **Example**: "Brightness +40 → Quick lift for dark image"
- **Limitation**: Doesn't distinguish shadows from highlights - lifts everything equally
- **Better alternative**: Use Curves/Levels/Exposure for proper control

**Contrast**
- **What it does**: Increases separation between light and dark values
- **When to use**: Flat, washed-out images
- **Key parameters**: Contrast multiplier (+0.2 to +0.5 typical)
- **Situation**: Low-contrast hazy image
- **Example**: "Contrast +0.3 → Increases tonal separation"
- **Limitation**: Global application to all regions
- **Better for mixed lighting**: CLAHE (local contrast)

**Exposure**
- **What it does**: Brightness adjustment respecting camera metering intent
- **When to use**: Underexposed RAW images, respect capture exposure intent
- **Key parameters**: Exposure stops (-2 to +2)
- **Situation**: Underexposed camera capture
- **Example**: "Underexposed by 1.5 stops → Exposure +1.5 → Recovers intended brightness"
- **Advantage over Brightness**: Respects camera's exposure intent, better highlight preservation
- **Best for**: RAW image correction, backlit subject recovery

**Levels**
- **What it does**: Three-point tonal adjustment with histogram visualization
- **When to use**: Clipping correction, tonal range expansion
- **Key parameters**: 
  - **Black point**: Darkest value (drag right to set minimum)
  - **Midtones (Gamma)**: Middle brightness (0.1-2.0 range)
  - **White point**: Brightest value (drag left to set maximum)
- **Situation**: Clipped blacks/whites, unused tonal range
- **Example**: "Histogram shows data 30-200 (not using 0-255 full range) → Levels: Black point to 30, White point to 200 → Expands to full 0-255 → Maximum tonal range"
- **Workflow**:
  1. Drag black point right until histogram starts at zero
  2. Drag white point left until histogram ends at 255  
  3. Adjust gamma slider for overall brightness balance
- **Advantage**: Histogram visualization guides decisions
- **Use when**: Need to see and correct clipping

**Curves**
- **What it does**: Unlimited control points for precise tonal mapping
- **When to use**: Maximum tonal control, shadow/midtone/highlight independent adjustment
- **Key parameters**: Multiple control points on curve (unlimited)
- **Situation**: Precise shadow recovery, highlight protection, complex tonal correction
- **Common applications**:
  - **S-curve**: Increases contrast (darken shadows, brighten highlights)
  - **Raised shadows**: Lift dark values only
  - **Pulled highlights**: Reduce bright values only
  - **Custom curve**: Precise multi-point correction
- **Example - Dark face detail**: "Add point at shadow region (x=40, y=40) → Drag up to (x=40, y=80) → Shadows lifted 2× → Face detail emerges without affecting midtones/highlights"
- **Example - Prevent clipping**: "Highlights at 250 (near clipping) → Add point at (x=250, y=250) → Drag down to (x=250, y=220) → Prevents highlight blowout"
- **Verdict**: Most sophisticated tone correction tool
- **Professional standard**: Forensic default for detail extraction

**Histogram Equalization**
- **What it does**: Automatically spreads tones across full 0-255 range (global)
- **When to use**: Flat image with underused tonal range, UNIFORM lighting only
- **Key parameters**: None - automatic algorithm
- **Situation**: Low-contrast image with uniform lighting
- **Example**: Flat document scan - "Histogram Equalization → Auto-spreads tones → Uses full 0-255 range → Improved contrast"
- **Limitation**: Global adjustment - CAN amplify noise, create unnatural look in mixed lighting
- **When NOT to use**: Mixed lighting (use CLAHE instead), high-noise images
- **Advantage**: Automatic - no parameters to adjust

**CLAHE (Contrast Limited Adaptive Histogram Equalization)**
- **What it does**: FORENSIC GOLD STANDARD - local contrast enhancement
- **When to use**: Dark faces, fingerprints, mixed lighting, local detail extraction
- **Key parameters**: 
  - **Block size** (50-200 pixels): Processing region size
    - 50-75px = small blocks (very local detail extraction)
    - 100-150px = typical balanced (most common)
    - 200px = larger blocks (gradual enhancement)
  - **Clip limit** (2.0-4.0): Enhancement strength limiter
    - 2.0 = conservative (prevents over-enhancement)
    - 3.0 = typical balanced setting
    - 4.0 = aggressive (maximum detail, risk artifacts)
- **Situation**: Dark facial features in backlit shot, fingerprints on varied substrates, low-contrast documents
- **Example - Dark face**: "CLAHE block 100px, clip 3.0 → ROI on facial region → Dark facial features enhanced without affecting bright background → Face detail visible"
- **Example - Fingerprint on wall**: "CLAHE block 75px, clip 3.5 → Ridge detail emerges despite wall texture variation"
- **Mechanism**: Block-based processing - each region gets independent histogram equalization with clip limit preventing noise amplification
- **Advantage over Histogram Eq**: Prevents noise amplification, preserves natural appearance, handles mixed lighting
- **When to use**: PREFERRED enhancement for forensic work
- **When NOT to use**: Uniform images (use Histogram Eq), very high noise (denoise first)

**Smart Adjust**
- **What it does**: Content-aware automatic local enhancement
- **When to use**: Mixed content (face + background), impression evidence, complex subjects
- **Key parameters**: Strength (0-2.0)
- **Situation**: Face with varied background, impression with substrate variation
- **Example**: "Smart Adjust strength 1.5 → Analyzes local texture → Enhances face differently than background → Natural result with minimal tweaking"
- **Advantage**: Minimal manual parameter adjustment
- **Use when**: Complex subject with varied detail distribution
