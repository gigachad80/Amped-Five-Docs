### 2.12 PRESENT MODULE (Output, Annotation, Redaction)
**Purpose**: Court-ready output, professional annotation, privacy protection

#### **Filters & Sub-Filters**:

**Compare Original**
- **What it does**: Before/after visualization for transparency
- **When to use**: Court presentation, demonstrate enhancement, admissibility documentation
- **Key parameters**: Display mode selection
- **Modes**:
  - **Side-by-side**: Horizontal or vertical split
  - **Interactive slider**: Blend between original/processed
  - **Overlay**: Opacity control blend
- **Situation**: Court testimony, demonstrate no fabrication
- **Example**: "Compare Original side-by-side → Left: dark original face → Right: CLAHE-enhanced face → Demonstrates enhancement without fabrication"
- **Legal importance**: Often required to prove no content fabrication
- **Best practice**: Include in all court-bound reports

**Spotlight**
- **What it does**: Highlights specific region with vignette fade
- **When to use**: Draw attention to evidence, professional presentation
- **Key parameters**: Spotlight shape (circle/rectangle), fade size
- **Situation**: License plate characters, facial features, evidence detail
- **Example**: "Spotlight circular region around license plate → Fades background → Visual attention drawn to plate"
- **Use**: Court presentation, professional highlighting

**Hide Selection (Redaction)**
- **What it does**: Privacy protection through redaction
- **When to use**: Third-party privacy, witness protection, bystander faces
- **Redaction types**:
  - **Blur**: Soft anonymization (recognizable as person, not identifiable)
  - **Pixelate**: Mosaic effect (stronger redaction)
  - **Black box**: Full opaque removal (information security)
- **Situation**: Witness faces, bystander vehicle plates, innocent party privacy
- **Example**: "Select witness face → Pixelate 16×16 → Face unidentifiable while preserving evidence context"
- **Legal requirement**: GDPR/data protection compliance
- **Use when**: Footage contains innocent bystanders requiring privacy protection

**Annotate (Comprehensive Annotation)**
- **What it does**: MOST VERSATILE presentation tool - comprehensive markup
- **When to use**: All court presentation, evidence highlighting, professional reports
- **Features**:
  - **Text**: Labels with font/size/color control
  - **Macros**: Auto-insertion of frame #, filename, date, time, metadata
  - **Shapes**: Arrows, boxes, circles, polygons, lines
  - **Magnification**: Picture-in-picture zoom window
  - **Redaction**: Blur or pixelate sensitive areas (integrated)
  - **Paths**: Trace trajectories, motion paths, measurements
- **Situation**: All professional evidence presentation
- **Example - License plate**: 
  - Add box around plate
  - Arrow pointing to characters
  - Text label "Plate ABC-1234"
  - Macro insert {FRAME_NUMBER} for documentation
  - Magnification window showing character detail
- **Example - Crime scene path**: 
  - Trace subject path with colored line
  - Add text timestamps at path points
  - Arrow showing direction of movement
  - Box highlighting entry/exit points
- **Professional use**: Standard for all court presentation
- **Always include**: Frame numbers, scale references, date/time stamps

**Add Logo**
- **What it does**: Agency branding overlay
- **When to use**: Professional output, agency identification, copyright
- **Key parameters**: Logo file, placement (corner), transparency
- **Situation**: Police/forensic lab output
- **Example**: "Add department logo bottom-right corner, 70% opacity → Professional branding"

**Add Text**
- **What it does**: Static text and macro insertion
- **When to use**: Labels, documentation, evidence identification
- **Features**:
  - **Static text**: Manual entry (case numbers, descriptions)
  - **Macros**: Auto-insertion
    - {FRAME_NUMBER}: Current frame count
    - {FILENAME}: Source filename  
    - {DATE}: Capture date
    - {TIME}: Capture timestamp
  - **Metadata**: Auto-insert from file properties
- **Example**: "Add text 'Case #2024-1234' + {FILENAME} + {FRAME_NUMBER} → Each frame shows: Case #2024-1234 | parking_lot_cam3.mp4 | Frame 1547"

**Add Shape**
- **What it does**: Geometric markup elements
- **When to use**: Highlight regions, measurements, evidence marking
- **Types**:
  - **Rectangle**: Region marking
  - **Circle**: Target/focus area
  - **Arrow**: Direction/pointing
  - **Polygon**: Complex region boundary
  - **Line**: Measurement, trajectory
- **Properties**: Color, line width, opacity, fill/outline
- **Example**: "Yellow box around license plate + red arrow pointing to characters + green line showing vehicle path"
